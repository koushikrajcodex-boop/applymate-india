How it works

One-click bulk import flow

1. Choose Pack

GitHub Actions → Bulk Scholarship Auto Import → choose source_pack.

2. Dry Run

Run dry_run=true first. It checks discovery without writing Firestore.

3. Real Import

Run dry_run=false. Records import as draft/review and duplicates are skipped.

Required once: add FIREBASE_SERVICE_ACCOUNT_JSON and FIREBASE_PROJECT_ID in GitHub Actions secrets. Without that, dry-run works but Firestore import cannot happen.
